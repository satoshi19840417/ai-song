# Propose Lyric Directions

歌詞本文をいきなり作らず、まず方向性の比較案を3系統で提示する。

## Workflow

### 1. 方向性を確認する

最初に次を短く確認する。既に指定済みなら再質問しない。

- 曲の目的: どんな場面で使う曲か
- NG条件: 避けたいテーマ、語彙、表現
- 固定条件: ジャンル、BPM感、歌い手像、尺
- 比較観点: 何を重視して選ぶか（刺さりやすさ、意外性、再現性）

### 2. 過去曲の傾向を抽出する

以下の優先順で傾向を取得する。

1. `my-remotion-01/public/suno_PJ/Suno/_trend_snapshot.md` を検証して利用
2. snapshot が無効（不存在/破損/TTL超過/世代不一致）なら `01_Songs` 直接走査へフォールバック

フォールバック時の入力:

- `my-remotion-01/public/suno_PJ/Suno/_index.md`
- `my-remotion-01/public/suno_PJ/Suno/01_Songs/*.md`

返答には必ず `参照元: snapshot` または `参照元: fallback` を明示する。

分析軸:

- 主題（何を歌うか）
- 視点（一人称/二人称、語り口）
- 感情曲線（抑制型/爆発型など）
- 語彙・比喩（抽象/具体、季節語、色、距離）
- 構成傾向（サビ先行、ブリッジ重視など）

データ不足時は、代表曲を3曲以上提示してもらってから進める。

### 3. 3パターンを固定順で提案する

必ず次の順で、3案を同時に出す。

1. 傾向準拠パターン
2. 意外性パターン
3. 世間トレンド準拠パターン

#### 1) 傾向準拠パターン

- 過去傾向の主要要素を維持する
- 安定して「あなたらしさ」が伝わる案にする

#### 2) 意外性パターン

- 傾向との接点を1つだけ残す（完全断絶にしない）
- それ以外の2軸以上を意図的に外す
- 「実験的だが成立する」ラインで提案する

#### 3) 世間トレンド準拠パターン

- 個人傾向より市場の現在性を優先する
- 「最近の流行」が求められる場合は最新情報を確認し、日付付きで要約する
- 最新情報を取得できない場合は、仮説ベースであることを明示する

## Output Format

次の形式で提示する。

| パターン | 方向性要約 | 主題/視点 | 感情曲線 | サビのフック案（1〜2行） | 向いている編曲 |
|---|---|---|---|---|---|
| 1. 傾向準拠 | ... | ... | ... | ... | ... |
| 2. 意外性 | ... | ... | ... | ... | ... |
| 3. 世間トレンド準拠 | ... | ... | ... | ... | ... |

表の後に以下を必ず添える。

- 比較コメント: 3案の違いを2〜4行で要約
- 推奨選択ロジック: 目的別に「どれを選ぶとよいか」を明示

## Selection Step

最後に `1 / 2 / 3` の選択を促し、選ばれた案のみ詳細化する。

詳細化時の最小出力:

- セクション構成（Verse/Chorusなど）
- キーフレーズ候補 4〜8本
- 冒頭8行のドラフト

フル歌詞作成を求められたら `compose-lyrics` に接続して続行する。

## Quality Bar

- 3案の差分を明確にする（言い換え3案を禁止）
- 既存楽曲や著名フレーズの模倣を避ける
- 指定されたNG条件を厳守する
